步骤二、根据你的想法来设计提示词
-
（1）我的第一版提示词如下，等ai自己跑完后可以进行预览：
""
设计一套能够全面展示Injective区块链核心特色的卡片组，要求如下： 
 1. 卡片内容需准确体现Injective作为高速且具备互操作性的第一层区块链的定位，突出其专为构建顶级Web3应用而优化的核心价值主张。 
 2. 卡片组应包含6张卡片，分别对应以下关键特色： 
    - 高速高性能的链上交易处理能力 
    - 强大的跨链互操作性解决方案 
    - 完整的去中心化基础设施组件 
    - 多智能合约环境支持（CosmWasm和EVM） 
    - 开发者友好的即插即用型工具套件 
 3. 每张卡片设计需包含： 
    - 醒目的特色标题 
    - 简洁明了的功能描述（不超过40字） 
    - 相关的视觉图标或示意图 
    - 关键技术参数或优势说明 
 4. 整体设计风格要求现代、科技感强，符合Web3行业审美，色彩系统应与Injective品牌调性一致。 
 5. 卡片尺寸统一，支持在数字屏幕和印刷媒介上清晰展示，确保信息层次分明，易于快速阅读和理解。
""

（2）为了加入交互效果，进行修改，第二次的提示词如下：
“再完善一下，点击卡片能进行交互就更好了”
等待ai自己跑完后可以进行预览。此时项目就做的差不多了

（可选项)将项目推送到GitHub并部署到网络上  
-
1、将代码上传到GitHub上
- 访问GitHub仓库
- 确保仓库处于 main 分支（通常是默认分支）
- 在仓库主页点击绿色的 <> Code 按钮
- 选择 Upload files 选项
- 拖拽文件夹到上传区域
- 填写 Commit changes 信息，例如："Add Injective cards with Ninja Break theme"
- 点击 Commit changes 按钮
2、启用GitHub Pages
- 在仓库主页点击顶部导航栏的 Settings
- 在左侧菜单中选择 Pages
- 在 Build and deployment 部分：
- Source ：选择 Deploy from a branch
- Branch ：选择 main 分支
- Folder ：选择 /(root) 
- 点击 Save 按钮
3、等待部署完成
GitHub Pages会自动开始部署网站。部署完成后即可通过生成的URL来访问网站
